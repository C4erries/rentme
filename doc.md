1.2 Цели изменений (доработок) системы

Цель
Критерии достижения
Разработка веб-приложения «Маркетплейс аренды жилья с ML-прайсингом» — магазин для онлайн-аренды жилья.
Пользователь может зарегистрироваться и авторизоваться в системе, просматривать каталог жилья, бронировать, писать отзывы и общаться в чате. 
Обеспечение удобного и безопасного взаимодействия между клиентом и магазином.
Система корректно обрабатывает запросы, обеспечивает хранение персональных данных клиентов (e-mail, Имя) в защищённом виде.
Автоматизация бизнес-процессов маркетплейса.
Реализованы модули: регистрация и авторизация пользователей, каталог жилья, бронирования, система отзывов, управление пользователями и мессенджер.
Повышение удобства за счет предоставления рекомендуемой стоимости.
Реализован ml сервис прогнозирования цены жилья на краткосрочный (ночь) и долгосрочный период (месяц).

1.3 Область применения
Разрабатываемая система «Маркетплейс аренды жилья с ML-прайсингом» применяется для автоматизации деятельности маркетплейса по аренде жилья. Она предназначена для следующих категорий пользователей:
Клиенты — могут просматривать каталог, оснащенные расширенным поиском, бронировать жилье, оставлять отзывы, выставлять жилье на аренду, и пользоваться системой прогнозирования цены.
Админ – Имеет доступ к администрационной панели, метрикам и может писать любому пользователю.
Данный функционал может быть использован в следующих ИС:
E-commerce платформы — веб-системы для онлайн-аренды жилья (интернет-магазины).
Системы аналитики и маркетинга — для анализа поведения пользователей, эффективности сдачи аренды.
Учебные проекты и демонстрационные решения в области веб-разработки.
	
При реализации проекта были использованы следующие технологии:
Backend: Go (Gin), grpc
Frontend: HTML, CSS, TypeScript (React), адаптивная вёрстка (Tailwind)
СУБД: MongoDB, ScyllaDB
Прочее: Git, MiniIO (S3-хранилище), docker, docker-compose

2.1. Требования к функционалу

2.1.1 Регистрация и авторизация
Клиент
Для клиентов реализовать возможность создания личного кабинета со следующим списком данных: 
• Фамилия
• Имя
• Пол (мужской/женский)
• Дата рождения
• Почта
• Пароль
• Подтверждение пароля
• Согласие с обработкой персональных данных 

Админ/менеджер
Внести единственного админа в бд и сформировать для него логин и пароль, случайным образом. Админ будет иметь всего один аккаунт. В его кабинете реализовать возможность зарегистрировать нового менеджера или удалить имеющегося.
Данные, которые сотрудник предоставляет админу для регистрации:
• Имя
• Фамилия
• Электоронная почта
После создания нового сотрудника, ему предоставляется сгенерированный уникальный код (логин) и пароль для входа в систему.

Авторизация
Реализовать авторизацию клиента по e-mail и паролю и авторизацию менеджера и админа – по логину и паролю.

2.1.2 Функционал для каждого пользователя
Функционал неавторизованного клиента
Реализовать доступ клиенту к следующему функционалу (до авторизации):
• Главная страница
• Просмотр каталога книг
• Поиск
• Просмотр информации о книге
• Добавление и удаление книг из корзины
• Авторизация/регистрация 

1. Главная страница
В этом разделе клиент может ознакомиться с уникальными предложениями интернет-магазина, а также получить доступ ко всему остальному функционалу, такому как каталог, корзина, вход/регистрация, поиску. Сами предложения состоят из специализированной рекламы, бестселлеров и новинок. Поиск ищет по совпадениям буквосочетаний в названиях книг и переходит на отсортированный каталог, где показываются результаты.

2. Каталог
В этом разделе клиент может ознакомиться со всем имеющимся ассортиментом товаров в магазине, а также выбрать книгу по имеющимся предпочтениям и добавить её в корзину. Каждая книга в каталоге представляет название, автора, цену, и обложку. Также отсюда клиент имеет доступ к странице каждой книги, которая хранит всю информации о ней.
Фильтры, по которым можно отсортировать каталог, делятся на 3 вида: по жанрам, по авторам, по году выпуска. Сортировка подразумевает возможность использования сразу нескольких видов фильтров.
Сортировка по жанрам и авторам представляет собой мультивыбор из имеющимся в наличии. Сортировка по году выпуска представляет собой выбор временного интервала, в течение которого были выпущены данные книги.
Внизу каждой книги есть кнопка "добавить в корзину", которая помещает выбранную книгу в корзину.

3. Информация о книге
В этом разделе клиент имеет возможность получить всю информацию и выбранной книге, а именно: название, авторы, жанры, год выпуска, аннотацию и цену, поместить данную книгу в корзину.

4. Корзина
В этом разделе неавторизированный клиент имеет доступ просмотреть список выбранных им книг, удалить книгу из списка, выделить необходимые ему книги и выбрать их количество, в результате этих выделений можно узнать стоимость данной покупки.
В случае попытки оформить заказ выносится ошибка: "Оформление заказа доступно только зарегистрированными пользователями". 
5. Авторизация/регистрация
Описание в пунте 2.1.1 


Функционал авторизированного клиента
Авторизированный клиент имеет доступ ко всем функциям, которые были реализованы для неавторизированного пользователя, дополнительно необходимо реализовать:

• Профиль
• Оформление заказа
• Пополнение баланса
• Просмотр заказов

1. Профиль
В данном разделе пользователь имеет возможность отредактировать свои личные данные, удалить аккаунт, посмотреть список заказов, пополнить баланс. / Редакции можно подвергнуть все личные данные: фамилия, имя, возраст, пол. А также почту и пароль для входа. Все ТТ сохраняются как в регистрации клиента (п. 4.1). При редактировании пароля и почты запрашивается подтверждение.
При попытке удаления аккаунта запрашивается подтверждение. После подтверждения пользователь переходит на главную страницу в качестве незарегистрированного пользователя. Аккаунт невозможно восстановить. 

2. Оформление заказа
Оформление заказа происходит на странице "Корзина" (п. 5.4) с помощью кнопки "Оформить заказ". Если корзина пустая, выносится ошибка: "Корзина пустая". Если баланс клиента меньше суммы заказа выносится ошибка: "На вашем счете недостаточно средств. Пополните баланс". После оформления заказа книги, участвующие в заказе, удаляются из корзины, баланс уменьшается на сумму заказа, в списке заказов появляется новый заказ со статусом "Оформлен".

3. Пополнение баланса
Пополнение баланса происходит на странице профиля, предусмотренные поля:

• Номер карты
• Срок действия
• CVC/CVV-код
• Сумма пополнения

4. Просмотр заказов
Список заказов находится в профиле и раделяется на 2 категории: актуальные и завершенные. Актуальные содержат заказы со статусом "оформлен" или "готов к получению". Завершенные содержат заказы со статусом "получен" или "отказ". Каждый заказ содержит в себе информацию: код заказа, дату оформления, статус, список книг в заказе, его сумма.

Функционал менеджера
Реализовать для менеджера доступ к следующим страницам и функциям:
• Профиль
• Изменение статуса заказа
• Редактирование данных книги
• Добавление книги
• Удаление книги
• Взаимодействие с тегами: добавление и удаление


1.  Профиль
Профиль менеджера представляет собой список всех активных заказов (со статусом "Оформлен" и "Готов к получению").

2.  Изменение статуса заказа
У каждого заказа, представленного в списке, менеджер имеет возможность изменить его статус. Возможные варианты: оформлен, готов к получению, отказ, получен.

3.  Добавление книги
Добавление книги происходит из каталога, предусмотрены поля:
• Название
• Обложка
• Авторы
• Жанры
• Издательство
• Год выпуска
• Аннотация
• Цена
Для тегов Авторы, Издательство и Жанры реализовать мультивыбор соответствующих тегов.

4. Редактирование данных книги
На странице книги менеджер имеет возможность перейти на форму редактирования книги. Редактирование возможно любых полей (обложка, название, годы выпуска, авторы, жанры, аннотация, цена).

5.  Удаление книги 
На странице книги менеджер имеет возможность удалить данную книгу. При удалении запрашивается подтверждение. После удаления книга восстановлению не подлежит. После согласия с удалением менеджер переходит в каталог.

6. Теги 
Вынос таких тегов как Авторы, Жанры, Издательство обусловлен целью избежать повторения одних и тех же тегов в разных вариациях написания. (Пример: Лев Толстой, Лев Николаевич Толстой, Л.В.Толстой и т.д.) 
Добавление тегов происходит из каталога. Предусмотрены поля:
• Название тега
• Вид тега
Вид тега представляет собой выбор между классами тегов: Жанр, Автор, Издательство.
Название тега - непустое текстовое поле.
Удаление тега происходит из каталога и является формой, содержащей поля:
• К удаляемого тега - выбор: Жанр, Автор, Издательство
• Выбор тега из списка выбранного вида тега \ После удаления тега его восстановление невозможно.

2.2 Требования к ролям
Система предусматривает 4 вида ролей: 
1. Неавторизированный клиент – частное лицо или компания, заинтересованная в покупке книг онлайн, которое не имеет личный кабинет на нашем сайте.
Возможности неавторизированного клиента:
• Зарегистрироваться/авторизоваться в системе
• Смотреть каталог
• Смотреть книгу
• Смотреть корзину

2. Авторизированный клиент - частное лицо или компания, заинтересованная в покупке книг онлайн, которое имеет личный кабинет на нашем сайте.
Возможности авторизованного клиента включают в себя все возможности неавторизованного клиента, кроме регистрации, и дополняет их данными функциями:
• Оформлять заказ
• Просматривать историю своих заказов
• Пополнять свой баланс
• Просматривать данные профиля и редактировать их
• Удалить аккаунт

3. Менеджер – сотрудник, занимающийся выжачей заказов в пункте выдачи заказов. 
Возможности менеджера:
• Войти в систему
• Просмотреть список заказов
• Просмотреть заказ и изменить его статус
• Просмотреть каталог
• Просмотреть книгу, редактировать ее данные
• Удалить книгу
• Добавить книгу
• Взаимодействие с тегами: добавление и удаление

4. Админ – сотрудник, имеющий наибольшее число прав, занимается управлением остальными сотрудниками.
Админ обладает всеми возможностями менеджера и дополнительно следующими функциями:
• Добавить менеджера
• Удалить менеджера
• Просмотреть список менеджеров

2.5 Нефункциональные требования
1. Производительность:
1.1 Система должна оперировать на высокой скорости и обеспечивать быстрый доступ ко всем функциональным возможностям.
1.2 Время ответа системы на запросы пользователя должно быть минимальным, не превышать 2 секунд для обеспечения удобства использования.
1.3 Поиск книг должен выполняться не более чем за 1 секунду при запросе по названию или автору.
2. Безопасность
2.1 Все данные пользователей (логины, пароли, платежная информация) должны храниться в зашифрованном виде.
2.2 Поддержка HTTPS на всех страницах.
2.3 Обеспечение своевременных обновлений и исправлений уязвимостей, включая обновления веб-страницы и серверного программного обеспечения
2.4 Соблюдение законодательства о защите персональных данных должно включать шифрование данных и ограничение доступа к конфиденциальной информации только уполномоченным лицам.
3. Масштабируемость
3.1 Архитектура должна позволять горизонтальное масштабирование.
3.2 Возможность расширения функционала.
4. Доступность
4.1 Система должна корректно работать в популярных веб-браузерах Google Chrome, Yandex, Mozilla Firefox, Microsoft Edge, Safari и Opera.
4.2 Пользователи должны иметь одинаковый опыт использования системы независимо от выбранного браузера.
5. Юзабилити
5.1 Интерфейс должен быть интуитивно понятным и адаптированным под пользователей разного уровня подготовки.
5.2 Время обучения новому пользователю (освоение основных функций) - не более 5 минут.
5.3 Поддержка адаптивного дизайна для корректного отображения на ПК, планшетах и смартфонах.
6. Поддерживаемость и сопровождаемость
6.1 Работа должна быть документирована.
6.2 Использование контроля версий(git).
6.3 Возможность обновления без остановки всей системы.

7. Надежность
7.1 В случае возникновения сбоя система должна предоставлять возможность быстрого восстановления данных и восстановления работоспособности без значительных задержек.
7.2 Доступность системы должна составлять не менее 99,5% времени в месяц.
7.3 Система должна быть спроектирована с учетом возможных отказов и ошибок, чтобы минимизировать последствия и обеспечить продолжение работы без прерываний.
